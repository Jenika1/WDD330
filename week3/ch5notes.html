Objects

  • user-defined objects vs. built-in objects

Object Literals

Object = a self-contained set of related values and functions 

  • a collection of named properties that map to any JavaScript value

method = when a property’s value is a function.

Object Literals = an object that is created directly in the language by wrapping all its properties and methods in curly braces {}.

  • Also provide a useful way of organizing your code without polluting the global namespace.

** All objects are mutable at any time when a program is running. **

Creating Objects

  • const spiderman = {};
  • create an object using a constructor function (not recommended)
    • const spiderman = new Object();

Accessing Properties

  • access the properties of an object using the dot notation
    • superman.name
      << 'Superman'
  • bracket notation
    • superman['name']
      << 'Superman'

Computed Properties (introduced in ES6)

  • code can be placed inside square brackets and the property key will be the return value of that code.
    • const hulk = { name: 'Hulk', ['catch' + 'Phrase']: 'Hulk Smash!' };
    • << { name: 'Hulk', catchPhrase: 'Hulk Smash!' }

Calling Methods

Calling a method is the same as invoking a function, so parentheses need to be placed after
the method name.

Checking if Properties or Methods Exist

in - used to check whether an object has a particular property.

** objects can inherit properties from other objects **

hasOwnProperty() - used to check whether an object has a property that is its own, rather than one that has been inherited from another object

Finding all the Properties of an Object

for in - loops through all of an object’s properties and methods.

Object.keys() - will return an array of all the keys of any object that
is provided as an argument.

Object.values() - returns an array of all the object’s value.

Object.entries() - returns an array of key-value pairs. key-value pairs are returned in arrays.

Namespacing

object literal pattern - This is done by creating an object literal that serves as the namespace, then adding any values as properties of that object, and any functions as methods.

Namespace = place all your functions inside an object.

  • Now these functions need to be preceded by the namespace (object) to be invoked.

JSON

  • A lightweight data-storage format that is used by a large number of services for data serialization and configuration.
  • JSON is a string representation of the object literal notation
    • Property names must be double-quoted
    • Permitted values are double-quoted strings, numbers, true, false, null, arrays and objects
    • Functions are not permitted values

 parse()- takes a string of data in JSON format and returns a JavaScript object.

  • example:
    • JSON.parse(batman);
      << { name: 'Batman',
      'real name': 'Bruce Wayne',
      height: 74,
      weight: 210,
      hero: true,
      villain: false,
      allies: [ 'Robin', 'Batgirl', 'Superman' ] }

 stringify() - does the opposite, taking a JavaScript object and returning a string of JSON data.

  • example:
    • JSON.stringify(wonderWoman);
      << '{"name":"Wonder Woman","real name":"Diana Prince","height":72,
      "weight":165,"hero":true,"villain":false,"allies":["Wonder Girl",
      "Donna Troy","Superman"]}'
