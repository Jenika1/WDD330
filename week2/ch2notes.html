Programming Basics - Ch. 2

 **Be careful using. notation with integers. Javascript will confuse the dot with a decimal point.**

 

Other options: 

  •  Used to dots  5..Method()
  •  Put a space before the dot 5 .Method()
  • Write integers as a decimal 5.0.Method()
  • Place integers in parenthesis (5).Method()

 

Methods: (Number are always returned as string)

 

.toExponential() -  Returns numbers as a string in exponential notation. Example: 5..toExponential(); >>"5e+10"

 

.toFixed() -  Rounds number to fixed decimal place. Example: (3.141592).toFixed(2); >>"3.14"

 

.toPrecision() -  Rounds to a fixed number of significant figures.

 

.Number.isFinite() -  Returns true if the value is a number that isn't infinity, -infinity or NaN

 

Number(' ') -  Returns string to a number.

 

String() or .toString()

 

parseInt() -  Removes everything after decimal (doesn't use rounding)

 

parseFloat() -  Converts strings and 2 floating point decimal numbers.



% operator =  Calculates the remainder.

NaN = Not a number. 

Undefined = " I can't find a value for this" (set by default)

Null = 'No value'  Acts as a place holder 0 (manually set)



Lazy Evaluation -  Only check the minimum criteria that needs to be met

Soft equality -  "==" Javascript doesn't take into account the data type. So it will say that "5" string is the same as 5 number.

Hard equality - "===" Returns true if they are the same data type.

Inequality - "!=" or "!=="
